# 获取帮助

AxVisor 是一个开源项目，我们致力于为用户提供全面的支持。如果您在使用过程中遇到任何问题，可以通过以下方式获取帮助。

## 支持渠道

### 📖 文档优先
我们的[官方文档](/docs/introduction/overview)提供了详细的使用指南、API 参考和最佳实践。在提问前，请先查阅相关文档，这通常是解决问题最快的方式。

### 🐛 问题反馈 (GitHub Issues)
如果您发现了 bug 或有功能建议，请在 [GitHub Issues](https://github.com/arceos-hypervisor/axvisor/issues) 上提交。提交问题时请注意：
- 先搜索是否已有类似问题
- 使用清晰、描述性的标题
- 提供详细的重现步骤、环境信息和错误日志

### 💬 社区讨论 (GitHub Discussions)
对于一般性问题、想法分享或使用经验交流，欢迎在 [GitHub Discussions](https://github.com/arceos-hypervisor/axvisor/discussions) 中发起讨论。这里是：
- 分享最佳实践和解决方案的平台
- 与其他用户和开发者交流经验的地方
- 提出功能建议和参与技术讨论的空间

## 支持范围

### 社区支持
AxVisor 主要依靠社区支持，包括：
- 文档和教程
- 社区成员间的互助
- 开源贡献者的技术支持

### 商业支持
如果您有企业级需求或定制开发需求：
- 可以通过 GitHub Discussions 描述您的需求
- 考虑赞助项目或雇佣核心贡献者进行定制开发

## 常见问题

### 如何快速开始？
请参考我们的[快速入门指南](/docs/quickstart)，其中包含了在不同平台上安装和运行 AxVisor 的详细步骤。

### 如何报告安全问题？
如果您发现安全相关的问题，请不要在公开渠道披露。请通过 GitHub 的私有安全报告功能或联系项目维护者。

### 在哪里可以找到更多资源？
查看[社区资源页面](/community/resources)获取工具、教程和第三方集成。

## 获取帮助的最佳实践

1. **先搜索**: 在提问前，先搜索文档、GitHub Issues 和 Discussions
2. **提供详细信息**: 包括您的环境信息、复现步骤和错误日志
3. **使用正确的渠道**: Bug 报告使用 Issues，一般问题使用 Discussions
4. **保持礼貌**: 尊重社区成员，保持友好和专业的交流方式

我们感谢您对 AxVisor 项目的关注和支持！
